Servidor de Arquivos (FreeNAS)

Historico

Inicialmente a SEMED contava com um servidor de arquivos com o Windows XP com uma pasta compartilhada com os computadores do financeiros, mas esta sistema não possuia cabacidade suficiente para atender a demanda de acesso a pasta de varios usuarios simultaneamente, cousando probremas de acesso.

Para sanar esse problema foi subintituido o sistema pelo FreeNAS e criado a distribuição de varios pastas para os diverços setores.

Roteiro de Integração ao AD

  1. Ajustar o hostname: hostname e NetBIOS Name devem ser iguais;
    hostname: nas2 e Domain: semed-ni.intra | em Services - SMB - netbios: nas2.
  2. Ajustar os DNS: Apontar para os AD;
    Network - Global Configurarion - Nameserver 1: 172.15.0.3, Nameserver 2: 172.15.0.4, Nameserver 3 172.15.254.254.
  3. Ajustar NTP;
    Em System - NTP Servers adiconar o servidor de AD - 172.15.0.3.
  4. Integração do FreeNAs com o AD:
    em Directory Services - Active Directory.
    1. Criar o Usuario do freenas no AD: freenas.
    2. Domain Nome: semed-ni.intra
    3. Domain Account Name: freenas
    4. Domain Controller: adserver.semed-ni.intra
    5. Global Catalog Server: adserver.semed-ni.intra
  5. Teste no terminal
    1. wbinf -t
      checking the trust secret for domain SEMED-NI via RPC calls succeeded
    2. wbinf -t
      retorna lista de usuarios do dominio
    3. wbinf -g
      retorna lista de grupos do dominio
    4. getent passwd
      retorna lista de usuarios do dominio e locais
  6. Configura compartilhamento SMB;
  7. Criação do grupos dos compartilhamentos;
  8. Realizar teste em Windowns e no FreeNAS;
  9. Não permiteir que usuarios fora do grupo tenha acesso a pasta.

Erro smartd_daemon is not running

To disable SMART for USB devices, perform the following steps (old UI):
  1. Storage Tab
  2. Volumes View Disks
  3. Click on da0 (single boot device, might differ)
  4. Click edit and deselect the SMART service
